01 · Análisis semántico

Declara variables y luego una línea de uso (tipo nombre = valor; seguido de destino = expresión;). El analizador construye la tabla de símbolos, verifica que cada identificador usado esté declarado y comprueba la compatibilidad de tipos en cada operación, tal como lo haría la fase semántica de un compilador antes de generar código.

InEntrada

01Tabla de símbolos (ámbito local)

Las variables declaradas aparecerán aquí con su tipo.

02Verificaciones semánticas

Se listará, paso a paso, cada comprobación de declaración y de tipo.

03Árbol anotado con tipos

Cada nodo del árbol se etiquetará con el tipo que infiere el compilador.
02 · Tabla de tipos y símbolos

Declara variables como en Java: tipo nombre = valor;, una por línea. Se validará el tipo declarado contra el valor asignado y se construirán la tabla de tipos (reglas del lenguaje) y la tabla de símbolos (instancias concretas).

InEntrada

01Tabla de tipos

Se listarán solo los tipos usados en tu entrada.

02Tabla de símbolos

Cada identificador declarado aparecerá con su tipo, valor y dirección simulada.
03 · Generación de código

Reutilizando el mismo árbol sintáctico, cada operación binaria se traduce a código de tres direcciones (variables temporales) y luego a un pseudo-ensamblador con registros, tal como lo haría el backend de un compilador.

InEntrada

01Código de tres direcciones

t1 = ... t2 = ...

02Pseudo-ensamblador (registros)

LOAD, ADD, MUL, STORE por cada temporal generado.
04 · Gestión de memoria

Simula llamadas recursivas apilándose en tiempo de ejecución, igual que ejecutarFase() en Java: cada llamada empuja un registro de activación a la pila y crea un objeto en el heap; al retornar, el registro se destruye (POP) y el objeto queda disponible para el recolector de basura.
